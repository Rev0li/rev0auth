# FunFront

## Vision

Ce document est l'espace creatif frontend du projet: idees visuelles, themes UI, et micro-experiences pour garder du plaisir dans le dev.

## Direction artistique actuelle

- Style: glass + gradients doux + contrastes francs
- Typo: expressive et lisible
- Ambiance: pro mais fun, pas template standard

## Objectif des ecrans

1. Landing public
- Montrer la proposition de valeur en 5 secondes
- Orienter vers portfolio, CV, contact

2. Dashboard admin/user
- Voir en direct que les 2 cotes fonctionnent
- Donner des signaux clairs (OK/DOWN)

3. Zone membre
- Interface sobre et rapide
- Acces contenus selon role

## Checklist Fun Front

- Cohesion des couleurs
- Responsive mobile/desktop
- Temps de chargement court
- Feedback visuel clair sur les actions
- Aucune ambiguite sur les etats critiques

## Idees a tester

- Theme switch "studio" / "production"
- Mini animations d'etat (pulse vert, alerte rouge)
- Timeline d'activite dans le dashboard
- Widgets drag-and-drop pour admin

## Definition of fun done

Un ecran est "fun done" si:
- il est beau
- il est clair
- il est utile
- il donne envie de continuer a coder
